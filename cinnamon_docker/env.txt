# env.txt - Instructions and content for .env

This file explains the format of the `.env` file and provides an example.

Important: The `.env` file contains sensitive data (passwords). It is listed in `.gitignore` and should not be pushed to public repositories.

Example content (.env):

USER=admin
PASSWORD=docker
VNC_PASSWORD=docker

Usage:
- Edit the `.env` file and set secure passwords.
- Start the container with `docker compose up -d --build`.
